Your 404 pages are
blind spots for AI

When an agent hits a dead link, it gives up or hallucinates. agent-404 returns structured suggestions so agents recover gracefully. One script tag.

Add to your 404 page — that's it
<script src="https://agent404.dev/agent-404.min.js"
  data-site-id="your-site-id"
  data-api-key="your-api-key"
  defer></script>

How it works

Live pages

Beacon metadata

On every non-404 page, the script silently beacons URL, title, and headings to build your site index. Non-blocking, uses fetch with keepalive.

404 pages

Suggest alternatives

On 404 pages, the script fetches ranked suggestions and injects a human-readable list + a schema.org/ItemList JSON-LD block.

Fuzzy matching

Three-signal ranking

Path segment similarity (version-tolerant: v2→v3), Levenshtein distance for typos, and keyword overlap against titles and headings.

For agents

schema.org JSON-LD

Suggestions are injected as ItemList — a format AI agents and crawlers already parse. No custom namespace. No extra integration.

POST /api/suggest → response
{
  "deadUrl": "https://docs.acme.com/v2/auth",
  "suggestions": [
    {
      "url": "https://docs.acme.com/v3/auth",
      "title": "Authentication Guide",
      "score": 0.85,
      "matchType": "moved"
    }
  ],
  "jsonLd": { "@context": "https://schema.org", ... }
}

Use cases

📚

Docs version migrations

You shipped v3 of your API. Every agent trained on v2 links now hits 404s. agent-404 detects the version change and points them to the v3 equivalent.

🔗

URL restructures

Moved /blog/post-slug to /articles/post-slug? Instead of maintaining redirect maps forever, let the fuzzy matcher handle it.

🤖

AI agent recovery

LLM agents, RAG pipelines, and coding assistants follow links from training data. When those links rot, they hallucinate. Give them structured alternatives instead.

🔍

SEO & crawl health

Search engines discover your 404s. With JSON-LD suggestions in the page, crawlers find the right content instead of indexing a dead page.

404 detection

HTTP status codes aren't available in client-side JS. The script uses three fallback strategies (in order):

// 1. CSS selector you provide
<script ... data-404-selector=".not-found-page"></script>

// 2. Meta tag in your 404 template
<meta name="agent-404:status" content="404">

// 3. Title heuristic (contains "404" or "not found")

Quick start

1. Register your site
curl -X POST https://agent404.dev/api/sites \
  -H "Content-Type: application/json" \
  -d '{"domain": "your-site.com"}'

# → { "id": "abc-123", "apiKey": "key_..." }
2. Add the script tag to your site
<script src="https://agent404.dev/agent-404.min.js"
  data-site-id="abc-123"
  data-api-key="key_..."
  defer></script>
